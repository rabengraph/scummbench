AGENT BRIEFING

agent-game-harness

This site is designed primarily for AI agents. Read the mission, then open /game.

Mission

Navigate to /game and make progress in the adventure game running there. Use the symbolic state exposed by the runtime to decide what to do. Prefer acting on structured data over inferring from pixels.

Where the game lives

  • / — this briefing page.
  • /game — the playable game surface.
  • /status — optional debug view with the latest snapshot and recent events.

State inspection (read)

On /game, symbolic state is exposed via:

  • window.__scummState — the latest authoritative snapshot as a plain JavaScript object.
  • document.getElementById("scumm-state") — the same snapshot as JSON inside a <script type="application/json"> node.
  • Console events tagged [SCUMM_STATE] and [SCUMM_EVENT].
  • A visual object overlay and a debug state panel for human inspection.

Key snapshot fields:

  • camera.x — scroll offset for room-to-screen coordinate conversion.
  • haveMsg — text display state (0=none, 255=active, 1=ending).
  • verbs[] — includes dialog choices when a conversation is active.

Action API (write)

Agents can control the game via these functions on /game:

  • __scummClickVerb(verbId) — click a verb or dialog choice by ID.
  • __scummClickAt(x, y) — click at room coordinates.
  • __scummWalkTo(x, y) — walk ego to room coordinates.
  • __scummActionsReady() — check if action API is available.

Coordinates are in room/virtual-screen space (same as snapshot). To click an object, use its box center from roomObjects[]. Dialog choices appear in verbs[] when haveMsg != 0.

Operating rules

  1. Prefer symbolic state first.
  2. Use visual confirmation second.
  3. If telemetry and visuals disagree, trust the rendered game and report the discrepancy.
  4. Save frequently.
  5. Avoid repeating failed action loops.

Dev aid: mock mode

/game?mock=1 runs fake telemetry that matches the real v1 schema. Useful if the ScummVM runtime hasn't been built yet. Mock snapshots carry "mock": true. See claude/runbook.md for the full schema.
